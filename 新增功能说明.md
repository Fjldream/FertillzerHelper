# 水溶肥配方计算系统 - 新增功能说明

## 📋 功能概览

### 1. 自定义微量元素管理

**位置：** 原料管理 → 原料库表格 → 微量元素列

**功能：**
- 每种原料可添加多种微量元素（Ca、Mg、S、Fe、Mn、Zn、Cu、B、Mo、Co）
- 点击"+ 添加"或"✓ N种"按钮打开微量元素编辑弹窗
- 支持设置每种微量元素的百分比含量（0-100%）
- 自动保存到 localStorage，数据持久化

**预设数据示例：**
- 硫酸铵：S 24%
- 硫酸钾：S 18%
- 矿源：Ca 15%, Mg 3%
- 微量元素：Fe 2%, Mn 1%, Zn 0.5%, Cu 0.2%, B 0.5%, Mo 0.05%

### 2. 详细报告弹窗

**位置：** 含量计算/核价计算 → 结果区域 → "📊 查看详细报告（含微量元素）"按钮

**功能：**
- **主量元素分析**：N、P、K、Cl 的实际含量
- **微量元素分析**：所有微量元素的总含量和质量（kg）
- **原料明细表**：每种原料的百分比、用量、各元素贡献值
- 美观的卡片式展示，支持大屏弹窗

**计算逻辑：**
```
微量元素总含量 = Σ(原料百分比 × 原料中该元素含量)
微量元素质量 = 微量元素总含量 × 总重量 / 100
```

## 🔧 技术实现

### 数据结构
```javascript
// 原料对象
{
  name: '微量元素',
  price: 20000,
  n: 0, p: 0, k: 0, cl: 0,
  trace: {
    Fe: 2,
    Mn: 1,
    Zn: 0.5,
    Cu: 0.2,
    B: 0.5,
    Mo: 0.05
  }
}
```

### 核心函数
- `openTraceEditor(rawIndex)` - 打开微量元素编辑弹窗
- `saveTraceElements(rawIndex)` - 保存微量元素数据
- `getTraceSummary(trace)` - 获取微量元素摘要文本
- `openFormulaDetailReport()` - 打开含量计算详细报告
- `openPricingDetailReport()` - 打开核价计算详细报告
- `showDetailReport(...)` - 通用详细报告渲染

## 🚀 使用流程

1. **添加微量元素**
   - 进入"原料管理"
   - 点击原料行的"微量元素"列按钮
   - 在弹窗中输入各元素含量
   - 点击"保存"

2. **查看详细报告**
   - 完成"含量计算"或"核价计算"求解
   - 点击结果区域的"📊 查看详细报告"按钮
   - 查看完整的养分分析和微量元素含量

## ✅ 兼容性

- 自动兼容旧数据（无 trace 字段的原料自动添加空对象）
- localStorage 自动保存微量元素数据
- 导出/导入配置完整支持微量元素
